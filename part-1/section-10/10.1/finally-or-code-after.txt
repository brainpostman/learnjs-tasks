try {
  начать работу
  работать
} catch (e) {
  обработать ошибку
} finally {
  очистить рабочее пространство
}

try {
  начать работу
  работать
} catch (e) {
  обработать ошибку
}

очистить рабочее пространство

Первый вариант даёт преимущество при возникновении ситуаций, которые незамедлительно завершают выполнение программы (ошибка или return), с finally мы может "очистить рабочее пространство" даже в таких случаях, во втором случае очистки просто не произойдет и будут все соответствующие последствия от этого
